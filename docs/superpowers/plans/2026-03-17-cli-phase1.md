## Phase 2: CLI-Dashboard Parity (next branch after this)

Bring CLI to feature parity with the dashboard. Each is a separate task:

1. **`mecha config <name>`** — view bot config (`--json`, `--field`), edit (`--set key=value`, requires restart)
2. **`mecha sessions <name>`** — browse Claude Code conversation history (list, search, view detail, show full log). Reads `.claude/projects/` via dockerode exec.
3. **`mecha costs [name]`** — cost tracking: today/week/month totals, per-bot breakdown, `--json`. Reads `costs.json`.
4. **`mecha schedule <name> <subcommand>`** — manage schedules: ls, add, rm, pause, resume, run, history. Proxies to bot's `/api/schedule`.
5. **`mecha webhooks <name> <subcommand>`** — manage webhooks: ls, add, rm, history. Proxies to bot's `/api/webhooks`.
6. **Batch operations** — `mecha stop --all`, `mecha restart --all`, `mecha ls --status running`, `mecha ls --model opus`. Confirmation prompt for destructive batch ops.

New files: `src/commands/config.ts`, `src/commands/sessions.ts`, `src/commands/costs.ts`, `src/commands/schedule.ts`, `src/commands/webhooks.ts`

## Phase 3: Industrial Polish (follow-up branch)

The UX layer that makes the CLI feel professional:

1. **Color output** — `picocolors` for state-aware coloring (green=running, yellow=busy, red=error, cyan=names, dim=metadata). Respect `NO_COLOR` and `--no-color`.
2. **Spinners** — `ora` for operations >200ms: spawn, restart, rm, init.
3. **Shell completions** — `mecha completion bash|zsh|fish` command. Dynamic bot name completion.
4. **`mecha watch`** — live fleet dashboard in the terminal, refreshes every 5s. Shows name/state/model/uptime/cost/activity. Keyboard navigation (q=quit, r=refresh).
5. **Improved `mecha init`** — guided first-run: prerequisite checks (Docker, Node, API key), progress display, next-step suggestions.
6. **`mecha version`** — show all tool versions (mecha, claude-code, codex, gemini-cli, node, docker).
7. **Suggest next command** — after spawn/init/stop, print suggested next steps.

New files: `src/commands/watch.ts`, `src/commands/completion.ts`
